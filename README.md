### 🔐 WTMP - Who touched my phone
<img src="https://raw.githubusercontent.com/ojas1430/assets/main/Photos/BasicPics/wtmpapplogo.png" width="48" align="left" alt="WTMP logo"/>

**Mobile App** · Available on Google Play Store

**Key Highlights**
- Detects who tried to use your phone.
- Captures intruder selfies automatically.
- Shows unlock attempt history and logs.
- Locks selected apps with security protection.
- Helps protect your privacy and personal data.

`Kotlin` `Jetpack Compose` `Material 3` `MVVM` `Room Database` `CameraX` `Accessibility Service` `Overlay System` `App Lock` `Intruder Detection` `Coroutines` `StateFlow` `SharedPreferences` `MediaStore` `Permission Handling` `Local Storage` `Reactive UI` `Security Focused` `Lifecycle Aware` `Offline First`

[![GitHub](https://img.shields.io/badge/GitHub-ff007f?style=for-the-badge&logo=github&logoColor=white)](https://github.com/ojas1430/Who-Touched-My-Phone)
[![Play Store](https://img.shields.io/badge/Play%20Store-ff007f?style=for-the-badge&logo=google-play&logoColor=white)](https://play.google.com/store/apps/details?id=com.ojasx.whotouchedmyphone&hl=en_US)

<details>
<summary><b>📋 Project Details</b></summary>

**Short Description**  
Android security app that detects unauthorized phone access, captures intruder selfies, and locks selected apps using a PIN-protected overlay system.

**What the App Does**
- Detects when someone opens protected apps
- Shows a secure PIN lock screen
- Captures intruder selfies on wrong PIN attempts
- Saves unlock attempt logs with timestamps
- Lets users lock selected apps individually
- Works completely offline

**Tech Used**
- Kotlin · Jetpack Compose · Material 3 · Room Database · CameraX · Navigation Compose · Accessibility Service · SharedPreferences

**Features**
- App lock for selected apps
- Intruder selfie capture
- Unlock attempt history
- Secure PIN protection
- Recovery question support
- Real-time dashboard updates
- Modern Compose UI

**Engineering Highlights**
- Built a custom overlay lock screen using Accessibility Service
- Integrated CameraX for automatic intruder photo capture
- Stored logs and security data locally using Room
- Added SHA-256 PIN hashing for better security
- Managed app state using ViewModel and Compose state APIs
- Handled multiple Android permissions and system settings

**Challenges I Solved**
- Keeping the lock screen active over other apps
- Handling overlay permissions correctly
- Making CameraX work inside the security flow
- Preventing accidental unlock bypass during app switching
- Managing app lock states efficiently

</details>

### 📺 Playlist Progress
<img src="https://raw.githubusercontent.com/ojas1430/assets/main/Photos/BasicPics/playlistlogo.jpeg" width="48" align="left" alt="Playlist Progress logo"/>

**Mobile App** · Available on Google Play Store

**Key Highlights**
- Load any YouTube playlist using its link.
- View all videos from the playlist instantly.
- Mark videos as completed after watching.
- Automatically track your playlist progress.
- Helps organize and manage learning playlists.

`Kotlin` `Jetpack Compose` `MVVM` `ViewModel` `Firebase Auth` `Firestore` `Room DB` `Retrofit` `REST APIs` `Coroutines` `StateFlow` `Pagination` `Notifications` `YouTube API` `Google Sign-In` `Material 3` `GitHub`

[![GitHub](https://img.shields.io/badge/GitHub-ff007f?style=for-the-badge&logo=github&logoColor=white)](https://github.com/ojas1430/YT-Progress)
[![Play Store](https://img.shields.io/badge/Play%20Store-ff007f?style=for-the-badge&logo=google-play&logoColor=white)](https://play.google.com/store/apps/details?id=com.ojasx.eduplay)

<details>
<summary><b>📋 Project Details</b></summary>

**Short Description**  
Android app that lets users load any YouTube playlist, track watched videos, save notes, pin important videos, and monitor learning progress with a modern productivity-focused interface.

**What the App Does**
- Loads full YouTube playlists using playlist links
- Tracks completed and pending videos
- Lets users add notes and revision tags
- Allows pinning important videos
- Saves playlist progress automatically
- Supports reminder notifications
- Works even when internet is unavailable

**Tech Used**
- Kotlin · Jetpack Compose · Material 3 · Room Database · Firebase Auth · Firebase Firestore · Retrofit · DataStore · Coroutines & Flow · YouTube Data API · Lottie Animations

**Features**
- YouTube playlist importing
- Video progress tracking
- Notes and revision management
- Pinned videos system
- Google Sign-In support
- Email verification login flow
- Offline playlist caching
- Reminder notifications
- In-app video playback

**Engineering Highlights**
- Built playlist loading using YouTube Data API with pagination
- Preserved user progress even after refreshing playlist data
- Added offline support using Room and DataStore
- Integrated Firebase Authentication and Google Sign-In
- Used Firestore realtime updates for profile syncing
- Added notification reminders using AlarmManager
- Handled network changes and playback restrictions gracefully
- Managed app state using ViewModel, StateFlow, and Compose state

**Challenges I Solved**
- Preventing user progress from getting lost after playlist refreshes
- Handling offline mode when network/API fails
- Managing YouTube embed restrictions properly
- Supporting Android notification permissions on newer versions
- Keeping playlist and user state synchronized efficiently

</details>
